# JAVA 동시성 제어 방식
- - -
## synchronized
* 암시적 Lock
* 메서드에 synchronized 키워드를 사용하여 구현
>  장점 : 가장 간단하면서 쉽다.
> 
>  단점 : 하나의 스레드만 해당 메서드를 사용할 수 있어 병렬성이 떨어진다. 이는 성능 저하로 이어질 수 있다.
- - - 
## RentrantLock
* 명시적 Lock
* Lock 인터페이스를 사용
* lock() 메서드로 잠그고, unlock() 메서드로 잠금을 해제한다.
* ConcurrentHashMap과 혼합하여 성능을 높힐 수 있다.
> 장점 : 시작과 끝을 명시하기 때문에 여러개로 나누어서 작성할 수 있다.
> 
> 단점 : 개발자의 실수로 lock을 해제하지 않을 경우 문제가 생길 수도 있다. try-finally문이 추가되어 코드가 덜 간결해진다.
